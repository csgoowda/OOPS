# Most Important OOP Difference Tables

## Class vs Object

| Class          | Object           |
| -------------- | ---------------- |
| Blueprint      | Instance         |
| No memory      | Memory allocated |
| Logical entity | Physical entity  |

---

## Constructor vs Destructor

| Constructor        | Destructor           |
| ------------------ | -------------------- |
| Initializes object | Destroys object      |
| Called first       | Called last          |
| Can be overloaded  | Cannot be overloaded |

---

## Encapsulation vs Abstraction

| Encapsulation     | Abstraction              |
| ----------------- | ------------------------ |
| Hides data        | Hides implementation     |
| Security          | Simplicity               |
| Access specifiers | Abstract class/interface |

---

## Overloading vs Overriding

| Overloading          | Overriding      |
| -------------------- | --------------- |
| Compile-time         | Runtime         |
| Same class           | Parent-child    |
| Different parameters | Same parameters |

---

## Virtual Function vs Pure Virtual Function

| Virtual              | Pure Virtual           |
| -------------------- | ---------------------- |
| Has implementation   | No implementation      |
| Optional override    | Mandatory override     |
| Runtime polymorphism | Creates abstract class |

---

## Abstract Class vs Interface

| Abstract Class          | Interface                 |
| ----------------------- | ------------------------- |
| Partial abstraction     | Complete abstraction      |
| Can have normal methods | Only pure virtual methods |
| Can have data members   | Generally none            |

# Top 25 OOP Interview Questions

## Basics

### 1. What is OOP?

Using objects and classes to design software.

### 2. Why OOP?

Reusability, security, maintainability.

### 3. What is a class?

Blueprint for objects.

### 4. What is an object?

Instance of a class.

### 5. Class vs Object?

Blueprint vs Instance.

---

## Constructors & Destructors

### 6. What is constructor?

Automatically called when object is created.

### 7. What is destructor?

Automatically called when object is destroyed.

### 8. Can constructors be overloaded?

Yes.

### 9. Can constructors be virtual?

No.

### 10. Can constructors return values?

No.

---

## OOP Pillars

### 11. What is encapsulation?

Wrapping data and methods together.

### 12. What is abstraction?

Showing only essential information.

### 13. What is inheritance?

Acquiring parent class properties.

### 14. What is polymorphism?

One interface, many forms.

### 15. Encapsulation vs Abstraction?

Data hiding vs implementation hiding.

---

## Polymorphism

### 16. What is function overloading?

Same function name, different parameters.

### 17. What is function overriding?

Redefining parent function.

### 18. Compile-time polymorphism?

Function overloading.

### 19. Runtime polymorphism?

Function overriding + virtual functions.

---

## Advanced OOP

### 20. What is virtual function?

Function resolved at runtime.

### 21. Why virtual functions?

To achieve runtime polymorphism.

### 22. What is pure virtual function?

Virtual function assigned with = 0.

### 23. What is abstract class?

Class with pure virtual function.

### 24. What is interface?

Class with only pure virtual functions.

### 25. What is this pointer?

Pointer to current object.
